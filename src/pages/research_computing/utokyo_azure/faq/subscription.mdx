**サブスクリプション**は、Microsoft Azure における契約，課金の単位で，サブスクリプションの範囲で利用者は様々なサービスを利用することができます．UTokyo Azure においても，どのくらいの利用があったかや，寄付金総額を超過した場合にどの程度の負担をいただくかはサブスクリプション単位で計算します．また，サブスクリプションは管理権限の範囲でもあり，そのサブスクリプションで作成された各リソースの権限は継承されます．

![](img/02_subscription.png){:.center.border}

#### 全般
- UTokyo Azure においては、サブスクリプションは申請サイトから申請いただきます．それ以外の方法で作成されたサブスクリプションを利用することはできません．

#### 予算
- サブスクリプションには，事前にギフトから一定額の予算が割り当て済みです．その枠内では自由に Azure のクラウドサービスを利用可能です．
  - Azure の仕様や UTokyo Azure サポートの管理都合上，利用できないサービスが存在します．
- 割り当て予算を超過した場合は，本契約の最終年度（2028年度予定）にサブスクリプション毎にご負担いただきます．

#### 権限・ロール
- サブスクリプションは Azure クラウドサービにおける，各ユーザーがサービスを利用する上で最大の権限範囲になります．サブスクリプション
- 申請者のアカウントは、そのサブスクリプションに**所有者**権限として付加されます．
- サブスクリプションは共同管理が可能ですが，そのユーザーのアカウントを**共同作成者**としてサブスクリプションに権限付与する必要があります．付与手順の詳細は[FAQの権限付加の仕方](/src/pages/research_computing/utokyo_azure/faq/addrole)を参照ください．
- サブスクリプションに付与した権限は，そのサブスクリプションで作成されたリソースに継承されます．
